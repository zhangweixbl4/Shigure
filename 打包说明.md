# 打包说明

这份说明面向第一次接触 .NET / VS Code 的用户，按步骤操作即可把项目打包成可以运行的 `.exe` 文件。

#### 为什么要写这个打包说明? 

因为会被攻击, 前身冬月(Fuyutsui Tinkerer)用的人多了, 时间也久了, 就被外部势力所攻击, 导致冬月无法正常更新, 而Shigure的目标就是让每个人都有个自己的Shigure。

#### 建议每个使用的用户, 都能自己改名并打包一份自己`唯一的Shigure`。 

## 1. 安装 .NET SDK

本项目使用 `.NET 10`，需要安装 **.NET 10 SDK**，不是只安装 Runtime。

1. 打开微软官方下载页：

   ```text
   https://dotnet.microsoft.com/download
   ```

2. 找到 `.NET 10`，下载并安装 `SDK`。

3. 安装完成后，重新打开 PowerShell，输入：

   ```powershell
   dotnet --version
   ```

   如果能看到版本号，说明安装成功。

## 2. 用 VS Code 打开项目

1. 打开 VS Code。
2. 选择 `文件` -> `打开文件夹`。
3. 选择本项目文件夹，也就是包含 `Shigure.csproj` 的目录。

## 3. 修改项目名称

这样可以拥有自己唯一的软件, 防止军用科技对您进行攻击。


要把主界面、关于页、提示框里的 `Shigure` 一起改掉，使用 VS Code 的全局替换。

1. 在 VS Code 按 `Ctrl + Shift + H` 打开全局替换。
2. 在上面的搜索框输入：

   ```text
   Shigure
   ```

3. 在下面的替换框输入你想要的新名字，例如：

   ```text
   你想要的新名字(必须为英文)
   ```

4. 勾选 `区分大小写` 选项，快捷键是 `Alt+C`。这样只会匹配 `Shigure`，不会误替换 `shigure` 或 `SHIGURE`。
5. 确认后再点击 `全部替换`。

注意：不建议随便改文件名和文件夹名。项目文件仍然可以叫 `Shigure.csproj`，不影响打包。

## 4. 打包成 exe

在项目根目录打开 PowerShell，也就是 `Shigure.csproj` 所在目录，然后运行：

```powershell
dotnet publish .\Shigure.csproj -c Release -r win-x64 --self-contained true -p:PublishSingleFile=true -p:EnableCompressionInSingleFile=true -o .\artifacts\publish\win-x64
```

参数含义简单理解：

- `-c Release`：使用发布模式打包。
- `-r win-x64`：打包 Windows 64 位版本。
- `--self-contained true`：把运行需要的 .NET 组件一起带上，别人电脑不装 .NET 也更容易运行。
- `PublishSingleFile=true`：尽量打成单个 exe。
- `-o .\artifacts\publish\win-x64`：指定输出目录。

## 5. exe 文件在哪里

打包完成后，文件会在：

```text
artifacts\publish\win-x64
```

默认情况下，exe 名字来自 `Shigure.csproj` 里的 `AssemblyName`。

例如当前配置是：

```xml
<AssemblyName>Shigure</AssemblyName>
```

那么 exe 文件就是：

```text
artifacts\publish\win-x64\Shigure.exe
```

如果你把 `AssemblyName` 改成了 `Militech`，则文件会变成：

```text
artifacts\publish\win-x64\Militech.exe
```

## 6. 发给别人时要带哪些文件

不要只复制 exe。建议把整个文件夹一起打包发送：

```text
artifacts\publish\win-x64
```

因为程序还会使用这些运行数据：

```text
config
keymap
module
Assets
```

这些文件会在发布时复制到输出目录。最稳妥的方式是把 `win-x64` 整个文件夹压缩成 zip，再发给别人。

## 7. 常见问题

### 提示 dotnet 不是内部或外部命令

说明 .NET SDK 没装好，或者安装后没有重新打开 PowerShell。重新安装 .NET SDK 后，再打开一个新的 PowerShell 试一次。

### 打包成功但找不到 exe

检查这个目录：

```text
artifacts\publish\win-x64
```

exe 文件名不是固定的，它取决于 `AssemblyName`。

### 改名后图标或界面文字不对

先确认是否只改了 `AssemblyName`。如果使用了全局替换，建议重新检查这些位置：

```text
Shigure.csproj
UI\MainForm.cs
UI\StatusForm.cs
README.md
```

保存后重新运行打包命令即可。
